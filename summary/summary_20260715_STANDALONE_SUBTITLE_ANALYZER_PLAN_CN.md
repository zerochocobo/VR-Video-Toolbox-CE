# 独立字幕调试分析器开发研究与计划

日期：2026-07-15

## 结论

可以开发，而且现有“字幕调试分析器”很适合扩展为独立工具。建议保留当前 debug 会话模式，同时新增“媒体文件模式”：用户直接选择 MP4/MKV 等普通视频，工具自动发现同名 `.srt`、`.jp.srt` 及其他可选字幕；缺少 WAV 时允许先只看字幕，也可以按需从视频提取分析音轨并刷新波形。

这项扩展不需要 ASR 模型，只依赖项目已有的 FFmpeg、SRT 解析、soundfile/NumPy 和 WinMM 播放能力。

## 推荐入口与模式

分析器顶部增加“数据来源”选择：

- `Debug 会话`：保持现有逻辑，选择 `<name>_debug` 或其上级目录。
- `媒体文件`：通过文件选择器直接选择 MP4、MKV、MOV、AVI、WebM、M4V 等视频。
- 可选后续扩展 `字幕文件`：单独选择 SRT，在没有视频时只进行字幕时间轴检查；首版可以暂不实现。

当“字幕生成”页面没有选择目录时，打开分析器不扫描当前目录，直接等待用户选择 debug 目录或媒体文件。

## 普通视频加载流程

1. 用户点击“选择视频”，选择 `movie.mp4`。
2. 工具读取媒体时长，并搜索同目录下与视频 stem 匹配的字幕。
3. 字幕下拉框列出候选文件，例如：
   - `movie.srt`
   - `movie.jp.srt`
   - `movie.zh.srt`
   - `movie.en.srt`
   - 可选兼容 `movie.*.srt`
4. 默认选择优先级建议为：`movie.srt` > `movie.jp.srt` > 其他同 stem SRT。
5. 搜索可复用 WAV：
   - `<视频目录>/<stem>_debug/<stem>.wav`
   - 可选兼容 `<视频目录>/<stem>.asr.wav`
6. 如果没有 WAV，弹出：

   `缺少视频音轨 WAV 文件，无法生成音轨波形图，是否现在生成？`

7. 用户选择“是”：后台调用 FFmpeg 生成 WAV，完成后自动刷新波形和播放器。
8. 用户选择“否”：立即进入字幕浏览模式，不阻塞窗口。

## 没有 WAV 时的界面行为

即使没有 WAV，也可以利用视频时长和 SRT 时间戳显示：

- 字幕列表。
- 时间轴。
- 字幕区间底色与左右栅栏。
- 当前定位时间和进度条。
- 点击字幕条目后定位对应时间区域。

波形区显示空背景和居中提示：

`尚未生成音轨 WAV，无法显示波形和播放音频。点击此处生成。`

点击波形区时：

- 如果已有 WAV，执行正常 seek。
- 如果没有 WAV，再次询问是否生成。
- 用户确认后启动后台提取；取消则保持字幕浏览状态。

因此字幕与时间轴仍有时间关系，只是没有实际音频振幅和音频播放能力，不需要把整个分析器禁用。

## WAV 生成方案

### 文件位置

建议继续复用现有约定：

`<视频目录>/<stem>_debug/<stem>.wav`

优点：

- debug 生成和普通视频分析可以共用同一份 WAV。
- 再次打开视频无需重复提取。
- 不会在视频目录直接遗留 `.asr.wav` 临时文件。
- 后续若生成 raw/VAD/removed 文件，目录结构保持一致。

### 音频格式

- 16 kHz。
- 单声道。
- PCM WAV。
- 首版默认不降噪，保证波形代表原视频音轨；可在后续增加“原始/降噪分析音轨”选项。

### 执行方式

- 抽取一个不依赖 ASR 的公共函数，例如 `ensure_analysis_wav(video_path, wav_path, progress_callback, stop_event)`。
- 复用现有 FFmpeg 进程管理逻辑，但不要调用字幕识别模型。
- 在后台线程运行，避免冻结 Tk 主线程。
- GUI 显示“正在提取音轨”，并提供取消按钮或复用窗口级 stop event。
- 完成后验证 WAV 存在且可由 soundfile 打开，再切换到 `WAV_READY` 状态并刷新波形。
- 失败时删除不完整临时文件，并显示 FFmpeg 错误摘要。
- 建议先输出 `<stem>.wav.part.wav`，成功后原子替换为正式 WAV，避免下次误读半成品。

## 状态设计

建议把当前分析器从“默认一定存在 debug WAV”改成明确状态机：

- `EMPTY`：未选择数据源。
- `MEDIA_SELECTED`：已选择视频，正在发现字幕/WAV。
- `SUBTITLE_ONLY`：字幕和时长已加载，但没有 WAV。
- `WAV_GENERATING`：后台提取中。
- `READY`：字幕、波形和播放器均可用。
- `ERROR`：媒体探测、字幕读取或 WAV 提取失败。

按钮和交互根据状态启用：

- `SUBTITLE_ONLY`：字幕列表可用，播放按钮禁用，波形区点击触发生成提示。
- `WAV_GENERATING`：禁止重复生成，允许浏览字幕，显示进度/忙碌状态。
- `READY`：启用播放、seek、波形拖动和字幕跟随。

## 结构调整建议

### `tool_subtitle/debug_analyzer.py`

- 将“数据源选择”“字幕发现”“WAV 发现”与绘制逻辑分开。
- 支持 `load_debug_session(path)` 和 `load_media_file(path)` 两条入口。
- 波形绘制函数允许 `audio is None`，此时绘制字幕时间轴占位界面。
- 字幕下拉框的值从固定 final/raw/VAD/removed 扩展为带路径的数据模型；debug 模式仍可显示友好的轨道类型。

### `tool_subtitle/logic.py`

- 提取公共的分析 WAV 生成功能，避免普通视频模式伪造一次字幕生成任务。
- 增加媒体时长探测或复用现有 ffprobe 工具。
- WAV 生成不加载 Whisper、不写 SRT、不影响字幕生成缓存。

### 可选新增模块

- `tool_subtitle/analyzer_source.py`：负责普通媒体、debug 会话、字幕候选和 WAV 路径解析。
- 这样 GUI、媒体发现和 FFmpeg 逻辑可以独立测试，避免 `debug_analyzer.py` 继续膨胀。

## 字幕候选匹配规则

为避免把无关字幕列入下拉框，建议：

- 必须与视频基础 stem 相同或以 `stem.` 开头。
- 接受：`movie.srt`、`movie.jp.srt`、`movie.zh-CN.srt`。
- 排除 debug 诊断字幕：`.raw.srt`、`.vad.srt`、`.removed.srt`，除非用户切换到 debug 会话模式。
- 文件读取继续使用 `utf-8-sig`，并保留解析器的容错行为。
- 下拉项同时显示文件名，内部保存绝对路径，避免多个目录或同名轨道混淆。

## 性能要求

- 选择视频后的 ffprobe、目录发现可以快速完成；任何 FFmpeg 提取和 WAV 波形计算必须放后台线程。
- 保持当前局部波形视口，不提供“适合全长”和缩放按钮。
- WAV 完成后继续使用向量化 peak envelope 和可见区域绘制。
- 加载新媒体时通过 token/generation id 丢弃旧线程的迟到结果，防止快速切换文件后串数据。
- 不在主线程递归扫描整个目录树；普通媒体模式只扫描视频所在目录的同 stem 文件。

## 异常与边界情况

- 没有任何同名 SRT：仍可生成波形，但字幕列表为空，并提示选择外部 SRT。
- 有多个 SRT：默认选择优先轨道，用户可下拉切换，切换不重新生成 WAV。
- 视频没有音轨：FFmpeg 返回明确提示，保持字幕浏览模式。
- WAV 已存在但损坏：提示重新生成，使用临时文件原子替换。
- 视频或字幕含中文路径：所有 subprocess 参数使用参数列表/现有安全封装，不拼接 shell 字符串。
- 用户在生成过程中关闭窗口：设置 stop event，终止 FFmpeg，并清理 `.part.wav`。
- 视频发生变化但旧 WAV 仍存在：建议比较 WAV/视频修改时间；WAV 早于视频时提示重新生成。

## 分阶段开发计划

1. 抽离媒体源模型、同名字幕发现、WAV 路径规则和状态机。
2. 增加“Debug 会话/媒体文件”来源选择与视频文件选择器。
3. 实现普通视频的字幕下拉列表和无 WAV 字幕浏览模式。
4. 实现无 WAV 提示框、波形区二次触发生成和播放按钮禁用逻辑。
5. 实现后台 FFmpeg WAV 生成、取消、临时文件和错误处理。
6. WAV 生成完成后自动载入波形/播放器并保持当前字幕和时间位置。
7. 补充中英日 i18n、自动化测试和正式打包环境人工验收。

## 测试计划

- 直接选择 `movie.mp4`，自动列出 `movie.srt` 和 `movie.jp.srt`。
- 没有 WAV 时分别点击“是”和“否”。
- 选择“否”后字幕列表、字幕时间块和点击定位正常，播放按钮不可用。
- 无 WAV 状态点击波形区再次出现生成提示。
- 生成成功后无需重开窗口即可出现波形并播放。
- 生成失败、取消和关闭窗口时不留下半成品。
- 重新打开相同视频时复用现有 WAV。
- 视频更新后检测到旧 WAV 并允许重新生成。
- 无字幕、有字幕无音轨、多字幕、中文路径、长视频均能正常处理。
- debug 会话模式的 raw/VAD/removed 功能不回归。

## 验收标准

- 分析器能够脱离字幕生成流程，直接选择普通视频和同名字幕工作。
- 缺少 WAV 不阻止字幕浏览；用户可拒绝生成并稍后从波形区重新触发。
- WAV 提取期间 UI 保持响应，不加载 ASR 模型。
- 提取成功后自动刷新为完整波形、字幕区间、播放和同步定位模式。
- 现有 debug 分析能力全部保留。
