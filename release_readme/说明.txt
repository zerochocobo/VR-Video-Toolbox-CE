◆ 各功能会用到的外部工具
════════════════════════════════════════════════════════

VR 视频工具箱由多个独立小工具组成，不同功能需要的外部组件不同：

  - FFmpeg / ffprobe：常用的视频和音频处理基础组件。
  - Lada 或 Jasna：只在使用 AI 去马赛克时需要。
  - 语音识别、OmniVoice、ECAPA、pyannote、Bandit-v2 等模型：
    只在使用字幕、克隆语音或完整配音相关工具时需要。

┌─────────────────────────────────────────────────────┐
│  FFmpeg（常用视频/音频处理组件）                         │
└─────────────────────────────────────────────────────┘

1. 打开此网址：https://www.gyan.dev/ffmpeg/builds/
2. 找到「ffmpeg-release-full.7z」（或 .zip），点击下载。
3. 下载完成后，解压缩，进入解压出的文件夹，再进入「bin」子目录。
   里面有三个文件：ffmpeg.exe、ffprobe.exe。
4. 推荐有三种方法可以使用 ffmpeg.exe 和 ffprobe.exe：
   方法1：将这两个 .exe 文件 复制 到本程序 VR_Video_Toolbox.exe 的同级目录下
   方法2：将这两个 .exe 文件 复制 到 C:\Windows 或 C:\Windows\System32 文件夹里。
   方法3：解压缩之后配置Windows的PATH环境变量指向该目录即可

┌─────────────────────────────────────────────────────┐
│  Lada-CLI（AI去马赛克模型，需要N卡）                     │
└─────────────────────────────────────────────────────┘

1. 打开此网址：https://codeberg.org/ladaapp/lada/releases
2. 根据页面提示下载最新版本。
3. 推荐有两种方法可以使用lada-cli.exe
   方法1：将 lada-cli 文件夹（包括_internal目录） 复制到本程序的同级目录下，确保lada-cli.exe和VR_Video_Toolbox.exe 在同级目录
   方法2：解压缩后配置Windows的PATH指向这个文件夹，例如：D:\Lada\
4. 注意：Lada 需要 NVIDIA 显卡（越好越快），CPU 处理极慢不推荐。

┌─────────────────────────────────────────────────────┐
│  Jasna-CLI（AI去马赛克推荐平替）                         │
└─────────────────────────────────────────────────────┘

Jasna 是 Lada 的新一代维护分支版本，功能相同、与现有流程完全兼容。

1. 打开此网址：https://github.com/Kruk2/jasna/releases
2. 下载最新的 Windows 版并解压缩。
3. 推荐有两种方法可以使用jasna-cli.exe
   方法1：将 jasna-cli 文件夹（包括_internal目录） 复制到本程序的同级目录下，确保jasna-cli.exe和VR_Video_Toolbox.exe 在同级目录
   方法2：解压缩后配置Windows的PATH指向这个文件夹，例如：D:\Jasna\
4. 在主界面「AI去马赛克引擎」里选择「Jasna」或「Lada」，程序会自动记忆。
5. 注意：Jasna 同样需要 NVIDIA 显卡。


◆ 去马赛克工具
════════════════════════════════════════════════════════

只有需要去马赛克时才使用这一类工具。主界面提供多种去马赛克模式：

┌─────────────────────────────────────────────────────┐
│  【推荐】一键模式                                       │
└─────────────────────────────────────────────────────┘
  适用厂商：S1VR、MDVR、VRKM、IPVR 等大多数主流厂商。
  操作最简单，自动识别马赛克位置并处理。

  ▶ 如何判断是否适用？
    在 VR 设备中打开原始视频，观察底部的马赛克，
    如果马赛克呈【扇形 / 弧形】 → 直接使用"一键模式"。
    如果马赛克呈【正方形 / 网格】→ 请勾选"去马赛克前先转换成鱼眼视角"。

┌─────────────────────────────────────────────────────┐
│  选区直接裁剪模式                                      │
└─────────────────────────────────────────────────────┘
  适用于：普通平面视频，或者在普通屏幕上看马赛克就是矩形的情况。
  操作：手动框选马赛克区域 → 程序处理后覆盖回原位。

┌─────────────────────────────────────────────────────┐
│  选区 VR转平面 模式                                    │
└─────────────────────────────────────────────────────┘
  适用厂商：FSVSS、SAVR、URVRSP、CRVR、PXVR 等，以及部分 3DSVR 影片。
  
  ▶ 如何判断是否适用？
    在 VR 设备中观察底部马赛克，如果马赛克呈【正方形】
    但在电脑上看原始文件马赛克是【梯形/斜边形】→ 请使用此模式。
  操作稍复杂，建议先熟悉一键模式后再尝试。

不确定自己的视频属于哪种情况？
→ 点击主界面下方的「不知道马赛克样式？用放大工具查看」按钮辅助判断。


◆ 三个“鱼眼”选项分别是什么意思
════════════════════════════════════════════════════════

软件里有三个地方会出现“鱼眼”，普通用户按用途区分即可：

1. 一键模式："去马赛克前先转换成鱼眼视角"
   这是去马赛克用的处理开关。适合在 VR 眼镜里看起来像正方形/网格的马赛克，
   尤其是 SAVR、URVRSP 等中轴或底部区域马赛克。程序只是在内部临时把左右眼
   转成鱼眼视角来识别和修复，处理完会自动转回原本的 VR 投影；最终成品仍然是
   正常 VR 视频。

2. VR 拆分/合并工具：鱼眼拆分、鱼眼合并
   这是手动流程用的。鱼眼拆分会把原视频拆成左右眼鱼眼文件；鱼眼合并则用于
   你已经有修复好的左右眼鱼眼文件，需要先转回 VR 半球再合并成 SBS VR 视频。

3. VR 投影转换工具：VR半球 <-> 鱼眼
   这是单独改变视频投影格式的工具。只有确实需要生成鱼眼文件，或把鱼眼文件
   转回 VR 半球时才使用。如果源文件是左右眼同屏的 SBS 视频，请勾选双屏/SBS
   选项，让程序分别转换左右半边后再正确拼回去。

普通去马赛克优先用一键模式；除非你明确需要单独的鱼眼文件，否则不要先手动跑
投影转换工具。


◆ 批量字幕生成工具
════════════════════════════════════════════════════════

点击主界面的「日语批量字幕工具」可进入字幕生成功能。

此功能独立于去马赛克使用，不依赖 Lada，但需要：
  1. FFmpeg（同上）
  2. 语音识别模型文件（首次使用点"下载模型"按钮，自动下载至 models 目录）
  3. 【可选】NVIDIA 显卡加速：若检测到 CUDA 环境自动启用，速度可提升数十倍。
     若没有 N 卡，程序会自动回退 CPU 识别，速度很慢但可正常使用。


◆ 克隆翻译配音工具
════════════════════════════════════════════════════════

如果想把视频对白翻译成目标语言，并尽量保留原说话人的音色，请点击主界面的
「克隆翻译配音」。

当前克隆工具是引导式流程：

单人语音克隆
   适合单个视频或同一文件夹里只有一个人的声音。先转录并翻译，再抽取候选
   音色片段。候选列表可以试听原声、翻译预览和固定目标语言样本，确认后保存
   为 SPEAKER1。

多人语音克隆
   适合多人对白。请先选择说话人数，再为每个说话人选择候选、导入 WAV+TXT、
   设计音色、导出或复用参考音色。不想生成克隆语音的说话人，可以设为
   「保留原声」。

参考音色要求
   导入的参考 WAV 建议 3 到 10 秒，TXT 必须和语音内容一致，而且语言必须和
   翻译目标语言一致。

输出和回混
   确认参考音色后，会生成按时间线对齐的 <视频名>.si.wav，并同时生成
   <视频名>.si.duck.wav。进入「混音配音」后，压低原声模式输出 _SI.mp4；
   Bandit-v2 移除原始人声模式会保留音乐/音效背景、混入克隆语音，输出
   _DUB.mp4。DLNA 服务器也可以通过直播流 [SI] 直接混入同名 .SI.WAV，
   不一定需要先生成新的 MP4。

此功能需要 FFmpeg、语音识别模型、OmniVoice，以及和字幕翻译共用的翻译 API
配置。多人和完整配音流程可能还需要 OmniVoice ECAPA、pyannote 说话人分离、
Bandit-v2 等模型。正式使用前，请务必试听生成的 .si.wav / _SI.mp4 / _DUB.mp4。


◆ 常见问题
════════════════════════════════════════════════════════

Q: 程序提示"找不到 ffmpeg"？
A: 请确认 ffmpeg.exe 与本程序 .exe 在同一文件夹内。
   或者到系统环境变量 PATH 中添加 ffmpeg 的 bin 目录路径。

Q: 去马赛克效果差 / 没有效果？
A: Jasna 和 Lada 对矩形马赛克效果最好。部分厂商的马赛克形态特殊，
   AI 无法还原，这是模型本身的技术限制。
   可尝试切换引擎（主界面 → AI去马赛克引擎），或改用其他模式尝试。

Q: 字幕工具识别出来乱码 / 不是日文？
A: 程序默认识别日语。请确认视频本身包含日语语音。
   识别结果会以 .jp.srt 格式保存在原视频同目录下。

Q: 运行过程中报错 "Option 'ad' not found" 或 "Error initializing filter" 等？
A: 这通常是因为您的 ffmpeg 版本过低，或者是缺少组件的“精简编译版”。
   本程序使用了高级音频降噪和 VR 全景滤镜，请前往官方推荐地址下载并替换为
   最新的完整版（Full Build），建议版本在 6.0 以上。
   推荐地址：https://www.gyan.dev/ffmpeg/builds/ (选 ffmpeg-release-full.7z)

Q: 在哪里下载本程序的最新版？
A: 软件首页：https://github.com/zerochocobo/VR-Video-Toolbox-CE
